# Meta-Workspace Workflow

A meta-workspace coordinates work across repositories. It is not where application source code normally lives.

Use it for:

- company and project context
- repository registry
- architecture decisions
- SDD/spec files
- PR review drafts
- agent instructions and templates
- memory setup and handoffs

Do not use it for:

- feature branches inside application repositories
- vendored dependencies
- generated build artifacts
- secrets
