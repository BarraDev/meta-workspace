# Pull Request Work

Default mode: draft-only.

Before reviewing or changing a PR, collect:

- repository
- PR URL or number
- author
- title and purpose
- base branch
- head branch
- changed files
- user intent: review only, fix, re-review, or prepare response

Rules:

1. Do not post comments without explicit user approval.
2. Prefer checking out the PR into a worktree.
3. Review the diff and relevant surrounding code.
4. Verify findings with concrete file/line evidence.
5. Distinguish blockers from suggestions.
6. If asked to fix a PR, make changes in a worktree and run project tests.
