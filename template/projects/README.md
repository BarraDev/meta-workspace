# Projects Registry

This folder describes the projects and repositories coordinated by this meta-workspace.

Application source code should usually live in `../repos`, and implementation work should usually happen in `../worktrees`.

Use `../repos/<project-id>` for clean main checkouts and `../worktrees` for feature or PR worktrees by default.

To add a project interactively:

```bash
./scripts/new-project.sh
```
